  - 상담 기록의 `메모` 행에는 우측 삭제 버튼을 둔다. 현재는 local state에서 해당 메모 기록을 타임라인에서 숨기는 방식이다.
  - `앱 상담 내역`은 큰 카드가 아니라 상단 고객 정보의 `상담경로`가 앱일 때 붙는 작은 대화창 버튼으로 처리한다. 버튼 클릭 시 차선생 앱 상담 큐/대화내역 패널을 여는 자리다. 아래 운영 영역은 첫 줄 `상담 기록` 2/3 + `다음 일정` 1/3, 둘째 줄 `견적함` 2/3 + `서류함` 1/3로 정렬한다. 네 섹션 모두 헤더는 바깥에 가볍게 두고 본문만 얇은 border 박스로 감싼다. 헤더 제목 바로 오른쪽에는 원형 카운터 배지와 같은 크기의 원형 `+` 버튼을 붙인다. `+` 버튼은 텍스트 없이 aria-label로 의미를 둔다. `서류함` 카운터는 `0/4`가 아니라 수령/첨부된 개수 `0`만 표시한다. `견적함`는 견적서 스택과 고객에게 나간 견적서 원본 이미지/파일 첨부 자리다. `서류함`은 면허증/심사서류 업로드 공간의 MVP 자리다. `다음 일정`은 날짜/시간/유형/메모로 통화/할 일/재연락/견적/서류 업무를 생성하는 작은 스케줄러 카드다.
  - dotted 점은 아주 미세하게 유지한다. 배경 surface는 흰색(`#fff`)에 아주 미세한 dot만 얹고, floating card는 투톤감이 약하게 살아나도록 `#f7f7f6` 단일 surface로 둔다. 구분감은 색 차이를 크게 만들기보다 border/shadow 중심으로 잡는다.
  - 니즈 카드 내부 padding은 위/아래 영역 모두 `10px`로 타이트하게 맞췄다. 문의사항 위 구분선은 카드 외곽선과 같은 `#dededb`.

## Operating Rules

- 이사님이 의견을 물으면 바로 구현하지 말고 판단/추천 후 `적용할까요?`.
- 이사님이 `해줘`, `적용해`, `진행해`, `응`처럼 실행 지시하면 바로 구현.
- 작은 CSS/타이포 조정: 테스트는 마지막 묶음에서만.
- DOM/기능 변경: `bun run typecheck` 우선.
- 큰 레이아웃 변경: 필요할 때만 Playwright screenshot 1회.
- 장시간 UI 작업은 1~2시간 단위로 새 세션 전환 권장.
