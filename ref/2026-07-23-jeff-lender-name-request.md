# [차선생 CRM → 제프] support-matrix 응답에 금융사 표시명(`lenderName`) 추가 요청

작성 2026-07-23 · 관련 회신 `ref/2026-07-21-jeff-support-matrix-reply.md`

## 요약

`GET /api/external/quotes/support-matrix` 잘 쓰고 있습니다. 특히 **파라미터 없이 전량 반환** + 행이 그쪽 lender SSOT 순서로 온다는 계약 덕에, **별도 lenders API 없이도** 저희 금융사 목록이 제프와 어긋났는지 이 응답 하나로 대조할 수 있게 됐습니다(오늘 그 가드를 붙였습니다). 감사합니다.

요청은 **작은 것 1건**이고, **급하지 않습니다.** 다른 작업 하실 때 곁다리로 얹어 주시면 충분합니다.

---

## 배경 — 저희가 금융사 목록을 어떻게 들고 있나

CRM은 금융사를 `{ code, label }` 쌍의 **하드코딩 상수**로 가지고 있습니다(제프 목록의 미러).

```ts
{ code: "mg-capital",   label: "MG캐피탈" },
{ code: "bnk-capital",  label: "BNK캐피탈" },
{ code: "im-capital",   label: "iM캐피탈" },
// … 8사
```

실시간 fetch로 바꾸지 않는 이유가 있습니다(저희 쪽 사정):

1. `code` 유니온이 **컴파일타임 타입**이라 런타임 값으로는 만들 수 없습니다(딜러 조회·지원집합 호출의 타입 안전이 여기서 나옵니다).
2. `label`은 **저장된 견적에 박히는 값**입니다(`crm.quotes.scenarios[].lender`) — 과거 견적 표시가 이 어휘에 묶여 있습니다.
3. CRM은 제프 미취급 금융사를 **수기 전용으로 추가**할 수 있는 상위집합 구조라, 제프 목록으로 통째로 덮으면 그 확장점이 죽습니다.
4. 제프 API가 잠깐 죽어도 금융사 드롭다운은 떠야 합니다(계산·딜러·지원집합만 제프 의존).

## 요청 — `lenderName` 한 필드

현재 매트릭스 행은 `lenderCode`만 싣습니다. 그래서 **code 드리프트(추가/삭제)는 잡히는데 개명(label 변경)은 못 잡습니다.**

| 드리프트 | 지금 감지 | 비고 |
|---|---|---|
| 금융사 추가 | ✅ | 새 `lenderCode`가 응답에 등장 |
| 금융사 삭제 | ✅ | 기존 `lenderCode`가 응답에서 사라짐 |
| **금융사 개명** | ❌ | 응답에 표시명이 없음 |

그래서 행에 표시명을 **한 필드만** 얹어 주실 수 있는지 여쭙습니다.

```jsonc
{
  "lenderCode": "im-capital",
  "lenderName": "iM캐피탈",        // ← 이 한 줄(제프 lender SSOT의 표시명 그대로)
  "productType": "operating_lease",
  "leaseTermMonths": [12, 24, 36, 48, 60],
  "annualMileageKm": [10000, 20000, 30000]
}
```

- **기존 필드는 그대로**, 추가만 부탁드립니다(저희 파서는 모르는 필드를 무시하므로 배포 순서 제약 없습니다).
- 같은 `lenderCode`가 productType별 여러 행으로 오는데, `lenderName`은 **행마다 같은 값**이면 됩니다(중복 무방).
- 별도 `GET /api/external/catalog/lenders`를 새로 만드시는 편이 그쪽 구조에 맞다면 그것도 좋습니다. 저희는 **표시명을 어디서든 한 번 읽을 수 있으면** 됩니다.

## 우선순위 — 낮음

개명은 드물고, **개명돼도 계산은 안 깨집니다.** 저희가 제프로 보내는 건 `lenderCode`라서 API 호출은 정상 동작하고, 상담사 화면의 **표시명만 낡습니다.** 그래서 이 요청은 "있으면 좋은" 수준이고, 급한 작업 앞에 끼워 넣지 않으셔도 됩니다.

만약 **가까운 시일에 금융사 개명·추가·제외 계획이 있으시면 그때 한 줄 알려 주시는 것만으로도 충분**합니다 — 그 편이 서로 더 쌉니다.

## 참고 — 저희 쪽 가드 현황

- 런타임: 워크벤치가 매트릭스를 받을 때 `lenderCode` 집합을 저희 상수와 양방향 대조해 어긋나면 콘솔 경고 1회.
- 수동: `bun run check:lenders`가 이 엔드포인트를 직접 조회해 차이를 출력(어긋나면 exit 1).
- 둘 다 **화면 동작은 바꾸지 않습니다**(fail-open 유지) — 어긋남을 기록만 합니다.
