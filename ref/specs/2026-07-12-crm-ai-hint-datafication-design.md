## 배경 (실측 확정)

고객 목록 "AI 힌트" 버튼(hover 보라 말풍선)은 **이름만 AI인 미구현 기능**이다:

- `client/src/lib/customer-table.ts:64` `aiHintDisplayByCustomerId` — 5월 목업 고객 20명
  (`CU-2605-0001`~`0020`) 전용 **하드코딩 문장 테이블**(이사님 2026-05-19 설계: 문장형·핵심어 굵게).
- 그 외 고객은 `crm.customers.ai_summary` 폴백 — **이 컬럼을 채우는 코드가 저장소에 없다**
  (스키마 정의 외 참조 0). DB 값 20개는 데이터화(#154) 시드 정적값이고, 하드코딩이 우선이라
  실제 화면에 DB 값이 보이는 고객은 0명. 앱 유입 실 고객 2명(CU-2606-0001/0002)은 NULL →
  **빈 보라 말풍선**(값 없음 방어 부재 — 유슨생 실기 발견 2026-07-12).
- 소비처: 목록 hover 말풍선(주) + 목록 검색 문자열(`CustomerManagementPage.tsx:176`) +
  레거시 카드형 all 모드 셀(:791, 메뉴 미노출). 고객 상세에는 없음.

## 확정 결정 (유슨생 2026-07-12 — ②는 이사님 문장 교체라 사후 확인 공유 권장)

1. **힌트 정의 = "고객을 읽는 한 줄"(목업 20문장 역설계 스펙)**: 무엇을 원하나 + 무엇에
   민감한가 + 지금 뭘 우선해야 하나. 60자 내외 1문장, 핵심어 굵게 ~2곳. **진행 단계별 관점**:
   | 단계 | 담는 것 | 목업 예 |
   |---|---|---|
   | 상담/견적 | 니즈+민감 포인트 | "X3·GLC 비교 중, **중도해지·월 납입액·총비용**에 민감" |
   | 심사 | 리스크+선결 조건 | "**사업자 증빙**이 약해 **승인 금융사**를 먼저 좁혀야" |
   | 계약완료/출고 | 잔여 작업 | "계약 확정 건으로 **출고 안내·법인 서류**만 남음" |
   | 보류/이탈 | 이탈 사유+재접근 | "**가족 반대**로 취소 — **재컨택 명분** 정리 필요" |
   | 공통 판단 신호 | 가능성/결정 구조 | "**배우자 결정** 영향이 큼" |
   역할 분리(5/19 결정 유지): 상담 메모(nextAction) = 담당자의 "할 일" / AI 힌트 = "왜"의 요약.
2. **목업 하드코딩 테이블 폐기, AI 생성값으로 교체**(백필로 전 고객 채움 — 단일 소스 =
   `ai_summary` 컬럼).
3. **갱신 = 고객 데이터 실변경 시 자동 재생성**(일 배치 아님).
