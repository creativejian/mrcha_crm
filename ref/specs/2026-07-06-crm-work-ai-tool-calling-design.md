# 업무 AI 도구 호출(Function Calling) — 설계

("일단 임의대로 정하고 나중에 변경" — 아래 "이사님 결정 필요" 3건은 **잠정 결정 + 사후 컨펌 대상**으로 전환.

## 배경 / 문제

- 업무 AI는 RAG 단일 경로(임베딩 검색 → 근거 생성)라 **집계·조건형 질의가 원리적으로 불가** — "조건에 맞는 모든 고객 스캔·목록화"는 유사도 검색이 못 하는 일(코퍼스 확장으로 해결 안 됨).
- **빠른 질문 5종(이사님 제작, Topbar)** 이 이 부류다. #169 유사도 임계값 이전에는 top-8 조각으로 "그럴듯하지만 불완전한" 답을 지어냈고, #169 이후 전부 NO_HITS로 정직하게 죽음 → 제품 기능 복구 필요.
- 07-06 논의(유슨생): **B안 = Gemini Function Calling 직행** 채택. C안(버튼→전용 리포트 매핑)은 B 설계 안에 "버튼 결정론"으로 흡수.

## 아키텍처 (확정 방향)

1. **RAG 경로 불변 + 도구 추가**. `/ask`가 Gemini 요청에 도구 선언(`tools`)을 동봉 — 모델이 근거로 답할 수 있으면 지금처럼 답하고, 목록·집계형이면 `functionCall` 발화 → 서버 실행 → `functionResponse` 재호출 → 최종 답변. 기존 검색·임계값(#169)·근거 표시는 그대로.
2. **자유 SQL 생성 금지 — 화이트리스트 도구만**. 도구 = 미리 짠 drizzle 쿼리 함수 + zod 파라미터 검증(`src/lib/assistant-tools.ts` 신설). 전부 read-only. scope는 `resolveCustomerScope`를 도구 내부에서 강제(역할 scope 슬라이스와 접점).
3. **빠른 질문 버튼 = 결정론 라우팅**. 버튼 클릭 시 클라가 도구 힌트 동봉 → 서버가 Gemini `tool_config`(mode ANY + allowed_function_names)로 해당 도구를 강제. 자유 질문만 모델이 라우팅 — 버튼은 매번 같은 쿼리를 정확히 탄다.
4. **NO_HITS 개편**: "근거 0 && 도구 미호출"일 때만 고정 답변. 근거 0이어도 도구가 답하면 정상 답변.
5. **근거 표시 확장**: sources에 도구 결과 유형 추가(`{sourceType: "tool", ...}` — "어떤 쿼리를 돌렸는지" 답변 하단 표기). 클라 근거 목록 렌더 소폭 확장.
6. **스트리밍**: 도구 왕복 단계는 논스트림, 최종 합성 답변만 기존 SSE 타자기. streamAsk 앞단에 도구 루프 삽입(dbHold·하트비트 등 스트리밍 전제 3종 불변).
7. **골든 질문 회귀 가드**: 현재 잘되는 RAG 질문 세트(제임스 견적 목록·김지안 구매시기·프로필 조회 등 07-06 실기 통과분)를 테스트로 잠가 도구 도입 후에도 기존 경로·품질이 유지됨을 기계 검증. 오라우팅 방지 시스템 프롬프트 지침("검색 근거로 답할 수 있으면 도구를 부르지 마라") 포함.

## 이사님 결정 필요 → 07-06 잠정 결정(유슨생) — 사후 컨펌 대상

1. **"오늘 견적 보낼 고객"** → 잠정: **진행 상태 "견적" 단계 고객 ∪ 작성 중(draft, 미발송) 견적 보유 고객** — 각 사유 병기. (후보였던 "견적 관련 할일"은 category 어휘가 불안정해 보류.)
2. **"출고/정산 리스크"** → 잠정: **계약완료 단계 ∩ 7일+ 무활동**("출고 진행 중 활동 공백" 근사) — 출고/정산 데이터가 생기면 실행기 쿼리만 교체.
3. **버튼 문구·응답 표현** → 버튼 문구는 불변(이사님 소유). 응답 표현 잠정: 계약 가능성 순위 = 확정>높음>중간>보류>낮음(상위 20), 응답 지연 = 7/15/30일 버킷(확인필요/지연/장기방치 — 클라 관리 상태와 동일 임계, 신규·상담접수 제외), 오늘 처리할 일 = 미완료 할일(기한 급함/오늘)+오늘 일정(KST). "내가"의 담당자 필터는 역할 scope 슬라이스 전까지 전체 기준.

## 비용/리스크

- 질문당 Gemini 호출 최대 2~3회(도구 왕복) — 지연·비용 소폭 증가. 도구 미사용 질문은 기존과 동일 1회.
- 규모: 중형(스펙 확정 → 플랜 → subagent 구현, 1~2일급). 손대는 곳 = assistant-tools.ts 신설 · /ask 도구 루프 · NO_HITS 분기 · streamAsk 앞단 · 클라 근거 렌더 · 골든 세트 테스트.
