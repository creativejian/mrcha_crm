### 항목 17 — 미지원 기간·약정거리 워크벤치 게이트 (#306, 2026-07-21): 🟡 사후 공유

파트너(제프)가 미지원 약정거리·리스기간에서 값을 지어내지 않고 **미취급 거절**로 정리했고(그쪽 PR #73·#76·#78), CRM은 그 지원집합을 받아 **견적 워크벤치에서 못 고르게** 막았습니다. spec = `ref/specs/2026-07-21-crm-support-matrix-gate-design.md`.

- **적용**: 선택 금융사가 취급 안 하는 **리스기간은 회색(선택 불가)**, **약정거리는 목록에서 제거**. 실제로 기간이 막히는 건 **MG캐피탈뿐**입니다(12·24개월 미취급 — 나머지 4사는 5기간 전부 취급). 약정거리는 5사가 전부 달라 체감이 큽니다(예: 25,000km는 우리카드만).
- **🟡 행위 변경 — 자동 폴백**: 24개월을 골라둔 상태에서 금융사를 MG로 바꾸면 **60개월로 옮겨지고 안내 문구가 뜹니다**(약정거리도 동일, 20,000km로). 막아버리면 그 값이 선택된 채 남을 수 없어서인데, **상담사가 친 값을 시스템이 바꾸는 유일한 지점**이라 공유합니다. 대안(값은 두고 조회만 차단)도 가능하니 불편하시면 바꿉니다.
- **안 막는 것**: 산은·iM·농협과 장기렌트는 파트너가 아직 지원집합을 확정 안 해(미착수) 그대로 전부 열려 있습니다. **이 3사는 지금도 미지원 값에서 가까운 값으로 당겨 계산합니다** — 파트너가 확정하면 우리 코드 변경 없이 자동으로 막힙니다.
- **계산기(비교견적) 모달은 무접촉** — 거기는 전 금융사를 한 번에 조회하는 화면이라 기준 금융사가 없어 게이트가 원리적으로 불가하고, 지금도 미취급 사는 결과에서 조용히 빠집니다.

### 항목 18 — 상담신청 "삭제"(dismiss)를 상담사가 할 수 있는 게 맞나 (배치 13 A#1): 🟡 판단 요청

고객 상세 드로어의 상담신청 카드 X 버튼 이야기입니다. **동작 자체는 설계대로**입니다 — 앱 상담신청(`public.consultations`)은 앱 소유·불가침이라 CRM은 그 행을 지우지 않고 "숨김 기록"만 남기고, 그 숨김은 **CRM 뷰 전체**(상세 드로어 · 관리자 인박스 · AI 힌트 재료 · 업무 AI 조회)에 적용됩니다. 원래 그런 설계입니다.

여쭙는 건 **권한**입니다.

- **되돌리는 방법이 없습니다.** un-dismiss 경로가 서버에 아예 없어서(누가 숨겼는지 기록은 남습니다), 상담사가 한 번 지우면 관리자 인박스에서도 사라지고 아무도 되살릴 수 없습니다.
- **같은 배치의 #307과 기준이 어긋납니다.** 담당자 재배정은 "되돌릴 수 없는 조작이니 관리자·팀장만"으로 막았는데(고객 하드 삭제 #212와 같은 급), dismiss는 되돌릴 수 없으면서 **관리자 화면까지 영향을 주는데** 상담사에게 열려 있습니다.
- **현재 실도달은 0입니다** — staff 계정이 테스트용 1개뿐이고 담당 고객이 0건이라 드로어에 들어갈 고객 자체가 없습니다. 다만 실무 상담사에게 고객이 배정되는 순간 열리고, 그때 대상은 **43건**입니다(미처리 상담신청 81건 중 CRM 고객에 연결되고 담당자까지 배정된 것).
- 선택지: ⓐ현행 유지(상담사도 삭제 가능) ⓑ#307과 같이 관리자·팀장만 ⓒ상담사는 드로어에서만 숨기고 관리자 인박스는 유지(설계 변경이라 비용 큼). **기각이면 코드 변경 0입니다.**

### 항목 19 — 미지원 조건 자동 폴백이 **되돌아오지 않는 것** (배치 13 K1 부수): 🟡 판단 요청

항목 17에서 공유드린 자동 폴백(미취급 기간·약정거리를 고르면 지원값으로 옮기고 안내)의 후속입니다. **옮겨간 값은 금융사를 원래대로 되돌려도 돌아오지 않고, 되돌리는 쪽에서는 안내도 안 뜹니다**(조용한 손실).

- 스펙에 "비가역"이라고 정한 적은 없어서 의도된 동작이라 부르기 어렵고, 결정이 문서에 없는 상태입니다.
- 기간은 MG 하나뿐이라 체감이 작지만, **약정거리는 5사가 전부 달라서** "금융사를 갈아끼우며 비교"하는 실사용 흐름에서 왕복 2~3회면 처음 세팅한 약정거리가 조용히 20,000km로 굳습니다.
- 선택지: ⓐ현행 유지(스펙에 "비가역"을 트레이드오프로 명시) ⓑ되돌리기 — 폴백 직전 값을 기억했다가 다시 지원되면 같은 톤으로 복원 고지("BNK캐피탈로 바뀌어 약정거리 40,000km를 되돌렸습니다"). ⓑ는 순수 함수 1 + state 1로 폴백 경로 안에서만 도는 소형 변경입니다.
- 판단이 갈리는 지점: ⓑ는 "금융사만 바꿨는데 약정거리도 또 바뀐다"는 **두 번째 무음 변경**을 만듭니다. 항목 17 회신과 함께 보시는 게 좋겠습니다.

### 항목 9 — 계산엔진(솔루션 조회) = 파트너 API 연동 (#241, 2026-07-14): 🟡 **유슨생 주도 확정·머지 완료 — 사후 공유 묶음** → ✅ 종결(2026-07-17, 이사님 컨펌 완료)

1. **계산엔진 = 자체 구축이 아니라 파트너(제프 financial-dolim-solution) API 연동** — 제프 플랫폼이 계산 SSOT. 견적 워크벤치 계산기 버튼이 실동작(금융사 미선택 시 전사 일괄 조회 랭킹 모달 — 제프 UI 이식).
2. **금융사 어휘 = 파트너 8사로 교체**(운용리스 8/장기렌트 3) — 구 목업 어휘(우리금융캐피탈·하나캐피탈)는 저장 견적 표시로만 잔존. 미취급 금융사는 수기 작성(추가 어휘는 한 줄 확장 구조).
3. **금리 의미론 변경**: 금융사 표면금리 → **리스계산기 실질(내재) 금리**(앱 lease_calc RATE 역산 — 예: 표면 5.22% 조건이 실질 7.01%로 표시). 앱 리스계산기와 같은 공식.
4. **결과 4필드(반납/인수/출고 전/금리) = 읽기 전용 자동 파생**(수기 입력 불가) — 구 견적 재진입·재저장 시 과거 수기 값이 파생값으로 대체됨. 산식: 반납=월납입×기간+선수금 / 인수=+잔존가치 / 출고 전=보증금+선수금+기타비용.
5. **월납입 채움 = 제프 표시 규칙**(운용리스 100원 올림) — 원값은 시나리오 스냅샷(solution_raw)에 보존.

기각이면 되돌릴 지점: 금리 의미론(3)은 파생 공식 1곳 교체, 나머지는 스펙 개정 단위 원복. 운영 전제: CF Pages Production `PARTNER_QUOTE_API_URL` 등록 전까지 prod 계산기는 503 안내(의도된 fail-loud).
