## 🔴 미해결

### 7. 수동 관리 상태(manageStatusOverride) 영속화 여부 (2026-07-13, #227 조사에서 발견)

고객 상세 워크플로우 카드의 **관리 상태 수동 선택**이 **어디에도 저장되지 않는다** — React 메모리 전용(`App.tsx` 주석 "manageStatus는 컬럼 없음"). 상담사가 "정상"으로 바꿔도 새로고침하면 자동 파생값으로 되돌아가고, 다른 팀원·업무 AI는 애초에 못 본다. 김민준 프로토타입 UI가 #154 데이터화(배지 자동 파생) 때 영속화만 안 따라간 반쪽 상태 — **입력을 받고 조용히 버리는 UI라 사용자에겐 버그처럼 읽힌다.**

- **선택지**: ①**영속화 + 스누즈 의미론(추천)** — 컬럼 신설·수동 상태는 "다음 실활동(메모/할일/견적 등) 기록 시 자동 해제"되는 임시 주석. 만료 규칙 없이 영속만 하면 60일 방치된 수동 "정상"이 영원히 거짓말하는 부패 문제가 생겨서, 스누즈가 "배지=활동 파생" 불변식을 지키는 형태. 수동 "재문의" 마킹은 `recontacted` 컬럼에 쓰도록 통합(이중 소스 방지 + AI #227 반영 자동 승계) ②**수동 입력 제거** — 관리 상태는 순수 자동 파생만, 수동 판단은 상담 메모로(메모 작성이 활동 기록이라 배지도 자연 복귀) ③현상 유지(비추 — 입력 유실 착시 지속).
- 5/19 이사님 원설계는 관리 상태 = 기간 파생(0~6 정상 등)+재문의 예외 — 수동 축을 어디까지 인정할지가 제품 판단.
- 결정되면: ①은 마이그 1 + PATCH·목록/상세 배선 + AI 도구 한 줄 슬라이스, ②는 워크플로우 카드 입력 제거 소형 슬라이스.
